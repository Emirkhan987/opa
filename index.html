1.что такое work flow:

work flow это что то наподобие плана действий. Например та же домашка сначала ты смотришь на задания которые тебе задали. А потом ты распределяешь время на ее выполнение. ну а дальше ты просто решаешь их по очереди.

1.2 для чего нужен workflow.

он нужен чтобы не запутаться и не терять много времени. workflow помогает что и в каком порядке выполнять

2.что такое git fetch

git fetch это как обновление к твоему коду. с помощью него можно узнать что и где твои товарищи по коду изменили. но сам ты ничего не меняешь

2.2 для чего нужен git fetch

как я и говорил ранее git fetch нужен для того чтобы узнать о изменениях в коде которые сделали твои товарищи но без возможности что либо изменить

github pages нужен для того чтобы опубликовать свои файлы в github и он сам сделает из них сайт
